## The new ship-blocking bug

macOS 26.4.1 SIGKILLs Rust-produced binaries on first execution. Symptoms Owner observed today:

```text
~ ❯ curl -fsSL https://raw.githubusercontent.com/izhiwen/AiPlus/main/install.sh | sh
[install succeeds, prints version=v0.6.11]
~ ❯ aiplus
zsh: killed     aiplus           ← EXIT=137 = SIGKILL
```

**Root cause**:
- Rust on macOS produces binaries with `flags=0x20002 (adhoc, linker-signed)` — the linker auto-applies an ad-hoc signature.
- macOS 26.4 enforces a new kernel-level check that SIGKILLs `linker-signed` adhoc binaries written from the network (even **without** the `com.apple.quarantine` xattr).
- The same binary, after `codesign --force --sign - --timestamp=none`, becomes `flags=0x2 (adhoc)` (no linker-signed flag) and runs normally.

**Confirmed live**: Advisor reproduced on Owner's machine. Re-signing the binary fixed the kill. v0.6.12 binary downloaded from GitHub Releases assets fails identically; re-signing fixes it.

**Affects**: every fresh install of every version we've shipped to macOS 26.4 users. This is silent — install.sh prints PASS but the binary cannot execute. Critical for adoption.

## Fix to bundle into 0.6.13

Two complementary changes:

### Change 1 — `release.yml` re-sign step after build (PRIMARY fix)

In `.github/workflows/release.yml`, after the `Build release binary (native)` step on macOS, add:

```yaml
- name: Strip linker-signed flag (macOS 26.4 SIGKILL workaround)
  if: runner.os == 'macOS'
  run: |
    # macOS 26.4 SIGKILLs binaries with flags=0x20002 (adhoc + linker-signed).
    # Re-codesign with plain adhoc to drop the linker-signed flag → flags=0x2.
    BIN_PATH=target/${{ matrix.target }}/release/aiplus
    echo "Before re-sign:"
    codesign -dvv "${BIN_PATH}" 2>&1 | grep -E "flags|Format" || true
    codesign --force --sign - --timestamp=none "${BIN_PATH}"
    echo "After re-sign:"
    codesign -dvv "${BIN_PATH}" 2>&1 | grep -E "flags|Format" || true
    # Verify the signature is adhoc-only (NOT linker-signed)
    if codesign -dvv "${BIN_PATH}" 2>&1 | grep -q "linker-signed"; then
      echo "ERROR: linker-signed flag still present after re-sign" >&2
      exit 1
    fi
    echo "OK: linker-signed flag removed"
```

Place it AFTER `Build release binary (native)` (line ~73) and BEFORE `Download token-cost binary` (line ~79). The token-cost binary is downloaded from a separate release and may need its own re-sign step — verify and add if needed.

### Change 2 — `install.sh` defensive re-sign (BACKUP fix)

In `install.sh`, after the `cp "$BIN" "$INSTALL_DIR/aiplus"` line (~line 217), add:

```sh
# macOS 26.4 SIGKILLs adhoc + linker-signed binaries. Re-codesign with
# plain adhoc removes the linker-signed flag. Belt-and-suspenders: even
# if the release pipeline produces re-signed binaries, this guards
# against users running old install.sh against new releases.
if [ "$(uname -s)" = "Darwin" ] && command -v codesign >/dev/null 2>&1; then
  codesign --force --sign - --timestamp=none "$INSTALL_DIR/aiplus" >/dev/null 2>&1 || true
  if [ -f "$INSTALL_DIR/aiplus-token-cost" ]; then
    codesign --force --sign - --timestamp=none "$INSTALL_DIR/aiplus-token-cost" >/dev/null 2>&1 || true
  fi
fi
```

The `|| true` ensures install doesn't fail if codesign is missing or rejects.
