### Retry-once gate (mandatory before classifying any FAIL)

Before classifying any first-attempt FAIL as `_<RUNTIME>-BUG_` or `_CONTRACT-BUG_`:

1. Re-run the SAME (spec × adapter) once, identical command + env.
2. If retry PASSes → record as PASS-after-flaky-retry and continue.
3. If retry also FAILs → proceed to App C classification.

Rationale: OpenCode and other non-interactive runtimes have observed flakiness in G-AT-PROD-1 round 2; one-shot FAIL is not enough signal to bucket. Two-shot FAIL is.

Track per-adapter conformance run in `docs/proposals/v1.1-contract-impl-notes.md` as evidence trailer.

## Iteration 2 changes (2026-05-18, post first BLOCKED)

The first Phase B run produced `RE_CONFORMANCE_VERDICT=BLOCKED` at OpenCode #02 inject_memory (`_OPENCODE-BUG_` classification, hard RUNTIME_FAILURE). CEO followed the briefing's "STOP at first FAIL" rule correctly. The rule was over-strict.

### Diagnosis

- App C decision tree is designed to distinguish CONTRACT bugs from runtime bugs at exactly this moment. STOPping on `_<RUNTIME>-BUG_` discards the data needed to confirm v1.1 soundness across remaining 19/21 datapoints.
- App E "Additive document versions" clause explicitly says re-conformance FAILs are by definition `_<RUNTIME>-BUG_` not `_CONTRACT-BUG_`.
- Adapter behavior on the OpenCode #02 FAIL was CONTRACT-correct (returned `exit_status=RUNTIME_FAILURE, partial=true` per §2 + §10 failure-path requirements). The bug is in OpenCode the runtime, not in the adapter and not in CONTRACT.

### Briefing fixes (this iteration)

1. **STOP rule narrowed**: only `_CONTRACT-BUG_` stops Phase B. `_<RUNTIME>-BUG_` annotates evidence + continues.
2. **Retry-once gate** introduced: every first-attempt FAIL must be retried once before App C classification. Catches flaky runtimes (OpenCode in particular).
3. **Acceptance criteria relaxed**: 21/21 PASS is no longer the binding gate. Binding gate is `zero _CONTRACT-BUG_`. Final verdict can be PASS / PASS_WITH_RUNTIME_BUGS / BLOCKED.
4. **New verdict `FROZEN-WITH-KNOWN-ISSUE`** when zero `_CONTRACT-BUG_` but ≥1 `_<RUNTIME>-BUG_`.

### What CEO does on resumption

Resume from where Phase B stopped: rerun OpenCode #02 once (retry-once gate), then continue OpenCode #03-#07 and full Codex #01-#07. Don't restart from Claude Code — those 7 PASSes are preserved.

If OpenCode #02 retry PASSes → record as PASS-after-flaky-retry and continue.
If OpenCode #02 retry FAILs → record as `_OPENCODE-BUG_` (already classified) and continue.
